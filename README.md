# KickPlus

KickPlus is an AI-powered football skill analysis app. Upload a short video of a player executing a skill and receive detailed technical feedback — score, positive points, areas to improve, and a personalised exercise — generated by Claude.

Supports **Portuguese** and **English**. Password-protected for controlled access.

---

## Features

- Analyse three fundamental skills: **shot**, **pass**, and **ball control**
- Optional **player profile** (age, height, weight, dominant foot) for personalised feedback
- Bilingual interface (PT/EN) with language preference persisted across sessions
- JWT-based authentication — single password, 30-day token

---

## Running locally

### Prerequisites

- Python 3.9+
- `ffmpeg` installed on the system
- An [Anthropic API key](https://console.anthropic.com/)

### Setup

```bash
cp .env.example .env   # then fill in the values
./start.sh
```

`start.sh` creates a virtual environment, installs dependencies, and starts the server at `http://localhost:8000`.

### Environment variables

Create a `.env` file in the project root:

```
ANTHROPIC_API_KEY=your-anthropic-api-key
ACCESS_PASSWORD=your-chosen-password
JWT_SECRET=any-long-random-string
```

Generate a secure `JWT_SECRET` with:

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

## Running with Docker

```bash
docker compose up --build
```

The app will be available at `http://localhost:8000`. `docker-compose.yml` loads the `.env` file automatically.

## Infrastructure setup (AWS)

This section documents the one-time AWS setup required before CI/CD can deploy automatically.

### Prerequisites

- [AWS CLI](https://aws.amazon.com/cli/) configured (`aws configure`)
- Docker installed and running
- Replace `<ACCOUNT_ID>` with your AWS account ID throughout

---

### 1. Create the ECR repository

```bash
aws ecr create-repository --repository-name kickplus --region us-east-1
```

---

### 2. Create the IAM role for App Runner to pull from ECR

```bash
aws iam create-role \
  --role-name AppRunnerECRAccessRole \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "build.apprunner.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'

aws iam attach-role-policy \
  --role-name AppRunnerECRAccessRole \
  --policy-arn arn:aws:iam::aws:policy/service-role/AWSAppRunnerServicePolicyForECRAccess
```

---

### 3. Push the first image to ECR

```bash
aws ecr get-login-password --region us-east-1 | \
  docker login --username AWS --password-stdin <ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com

docker build -t <ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com/kickplus:latest .
docker push <ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com/kickplus:latest
```

---

### 4. Create the App Runner service

```bash
aws apprunner create-service \
  --service-name kickplus \
  --source-configuration '{
    "ImageRepository": {
      "ImageIdentifier": "<ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com/kickplus:latest",
      "ImageConfiguration": {
        "Port": "8000",
        "RuntimeEnvironmentVariables": {
          "ANTHROPIC_API_KEY": "<your-anthropic-api-key>",
          "ACCESS_PASSWORD": "<your-chosen-password>",
          "JWT_SECRET": "<your-jwt-secret>"
        }
      },
      "ImageRepositoryType": "ECR"
    },
    "AutoDeploymentsEnabled": false,
    "AuthenticationConfiguration": {
      "AccessRoleArn": "arn:aws:iam::<ACCOUNT_ID>:role/AppRunnerECRAccessRole"
    }
  }' \
  --instance-configuration '{"Cpu": "1 vCPU", "Memory": "2 GB"}' \
  --region us-east-1
```

Note the `ServiceArn` and `ServiceUrl` from the output.

Check the service status:

```bash
aws apprunner describe-service \
  --service-arn <SERVICE_ARN> \
  --region us-east-1 \
  --query 'Service.Status'
```

---

### 5. Create the GitHub Actions IAM user

Create a scoped IAM policy that only allows pushing to the ECR repo and triggering App Runner deployments:

```bash
aws iam create-policy \
  --policy-name KickPlusGitHubActionsPolicy \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [
      {
        "Sid": "ECRAuth",
        "Effect": "Allow",
        "Action": "ecr:GetAuthorizationToken",
        "Resource": "*"
      },
      {
        "Sid": "ECRPush",
        "Effect": "Allow",
        "Action": [
          "ecr:BatchCheckLayerAvailability",
          "ecr:InitiateLayerUpload",
          "ecr:UploadLayerPart",
          "ecr:CompleteLayerUpload",
          "ecr:PutImage",
          "ecr:BatchGetImage",
          "ecr:GetDownloadUrlForLayer"
        ],
        "Resource": "arn:aws:ecr:us-east-1:<ACCOUNT_ID>:repository/kickplus"
      },
      {
        "Sid": "AppRunnerDeploy",
        "Effect": "Allow",
        "Action": "apprunner:StartDeployment",
        "Resource": "<SERVICE_ARN>"
      }
    ]
  }'
```

Create the user and attach the policy:

```bash
aws iam create-user --user-name kickplus-github-actions

aws iam attach-user-policy \
  --user-name kickplus-github-actions \
  --policy-arn arn:aws:iam::<ACCOUNT_ID>:policy/KickPlusGitHubActionsPolicy

aws iam create-access-key --user-name kickplus-github-actions
```

Save the `AccessKeyId` and `SecretAccessKey` from the output — AWS will not show the secret again.

## CI/CD

Every merge to `main` triggers the GitHub Actions workflow (`.github/workflows/deploy.yml`), which:

1. Builds the Docker image
2. Pushes it to ECR (tagged with the commit SHA and `latest`)
3. Triggers a new App Runner deployment

### Required GitHub secrets

Add these under **Settings → Secrets and variables → Actions**:

| Secret | Description |
|---|---|
| `AWS_ACCESS_KEY_ID` | Access key for the `kickplus-github-actions` IAM user |
| `AWS_SECRET_ACCESS_KEY` | Secret key for the `kickplus-github-actions` IAM user |
| `APP_RUNNER_SERVICE_ARN` | ARN of the App Runner service (from step 4) |
